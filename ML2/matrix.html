    Confusion matrices ML2

    Another useful way to evaluate a model is to produce a confusion matrix. This is a matrix that displays the counts of true versus predicted classes, highlighting correct and incorrect predictions. It applies to binary classification and also generalises naturally to multiclass problems.

    scikit-learn provides a function to create a confusion matrix. Here is an expanded function to simplify the visualisation of this matrix.

    Imports

    from itertools import product

from matplotlib.pyplot import show, subplots

from numpy import asarray, newaxis

    Confusion matrix plotting function

    # FIXED WITH OUTPUT

def plot_confusion_matrix(y_test, y_pred, classes, normalize=False, ax=None):
    """
    This function prints and plots the confusion matrix.
    y_test (array)
    y_pred (array)
    classes (array)
    normalize (bool) Normalize the results (True), or show them as integer numbers (False).
    ax Visualization axis.
    The function is an adaptation of a scikit-learn example.
    """

    cm = confusion_matrix(y_test,y_pred)
    n_classes = len(classes)

    if normalize:
        cm = asarray(cm).astype('float32') /cm.sum(axis=1)[:, newaxis]

    if not ax:
        fig, ax = subplots()

    ticks = range(n_classes)
    ax.imshow(cm, interpolation='nearest', cmap='Blues')
    ax.set_xticks(ticks)
    ax.set_xticklabels(classes, rotation=90)
    ax.set_yticks(ticks)
    ax.set_yticklabels(classes)
    fmt = '.2f' if normalize else 'd'
    thresh = 3*cm.max() / 4
    cm_dim = cm.shape

    # Matrix indices:
    indices_a = range(cm_dim[0])
    indices_b = range(cm_dim[1])
    # Cartesian product of matrix indices:
    indices = product(indices_a, indices_b)
    fmt = '.2f' if normalize else 'd'

    for ind_a, ind_b in indices:
      label = format(cm[ind_a, ind_b], fmt)
      color = "white" if cm[ind_a, ind_b] > thresh else "black"
      ax.text(ind_b, ind_a, label, ha="center", color=color)
    ax.set_ylabel('True label')
    ax.set_xlabel('Predicted label')

    return ax

    Ideally, the diagonal fields of the confusion matrix will be as bright (or as large in value) as possible, while the off-diagonal fields will be as dark (or as close to zero) as possible. The diagonal entries represent correct classifications: true negatives in the top-left and true positives in the bottom-right. High values here indicate that the model is accurately predicting both classes.

    Contrastingly, the off-diagonal squares represent misclassifications. The top-right cell corresponds to false positives, where negative samples are incorrectly predicted as positive; while the bottom-left cell corresponds to false negatives, where positive samples are incorrectly predicted as negative. Low values in these cells indicate that the model makes few classification errors, which is the desired outcome.
